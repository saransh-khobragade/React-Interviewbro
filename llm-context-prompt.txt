# Repository Context: InterviewBro

I need your help with a React TypeScript project. Here's the complete context:

## Project Summary
**InterviewBro** is a production-ready React 19 application built with TypeScript, Vite, Tailwind CSS, and shadcn/ui components. It features a complete authentication system and user management dashboard with CRUD operations.

## Technology Stack
- **React**: 19.1.0
- **TypeScript**: 5.8.3 (strict mode)
- **Build Tool**: Vite 7.0.4
- **Styling**: Tailwind CSS 4.1.11 + shadcn/ui components
- **Forms**: React Hook Form + Zod validation
- **State**: React Context API
- **Package Manager**: Yarn 1.22.22
- **Code Quality**: ESLint (ultra-strict) + Prettier + Husky

## Project Structure
```
src/
├── components/
│   ├── auth/              # AuthPage, LoginForm, SignupForm
│   ├── dashboard/         # Dashboard, UserForm
│   └── ui/                # shadcn/ui components (button, card, input, label, table, sonner)
├── contexts/
│   └── AuthContext.tsx    # Global auth state (user, login, signup, logout)
├── services/

├── types/

│   └── user.ts            # User, CreateUserData, UpdateUserData
├── lib/

│   └── utils.ts           # Helper functions (cn for className merging)
├── App.tsx                # Main component with auth routing
├── main.tsx               # Entry point
└── index.css              # Global styles + CSS variables
```

## Key Features Implemented

### 1. Authentication System
- Login/Signup forms with email/password validation
- Session persistence via localStorage (key: `'user'`)
- Mock API calls with 1-second delay
- Protected routing: shows AuthPage (logged out) or Dashboard (logged in)
- AuthContext provides: `user`, `login()`, `signup()`, `logout()`, `isLoading`

### 2. User Management Dashboard
- Display users in responsive table
- Statistics cards: Total Users, New This Week, Updated Today
- CRUD operations: Create, Read, Update, Delete users
- Modal forms for add/edit user
- Delete confirmation dialog
- Data stored in localStorage (key: `'users'`)
- 3 mock users pre-populated: John Doe, Jane Smith, Bob Johnson

### 3. UI/UX
- shadcn/ui components (modern, accessible)
- Responsive design (mobile-first)
- Toast notifications (Sonner)
- Loading states throughout
- Dark mode ready (theme infrastructure exists)

## Type Definitions
```typescript
interface User {
  id: string;
  email: string;
  name: string;
  createdAt: string;
  updatedAt: string;
}

interface AuthContextType {
  user: User | null;
  login: (credentials: LoginCredentials) => Promise<void>;
  signup: (credentials: SignupCredentials) => Promise<void>;
  logout: () => void;
  isLoading: boolean;
}
```

## Data Flow
1. **Auth**: Login → AuthContext updates → localStorage saves → Dashboard renders
2. **Users**: Dashboard → userService CRUD → localStorage updates → UI re-renders
3. **Routing**: App.tsx checks AuthContext.user → renders AuthPage or Dashboard
