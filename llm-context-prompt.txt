# Repository Context: InterviewBro

I need your help with a React TypeScript project. Here's the complete context:

## Project Summary
**InterviewBro** is a production-ready React 19 application built with TypeScript, Vite, Tailwind CSS, and shadcn/ui components. It's an interview preparation platform that displays coding interview questions organized by categories (Data Structures, Low Level Design, High Level Design).

## Technology Stack
- **React**: 19.1.0
- **TypeScript**: 5.8.3 (strict mode)
- **Build Tool**: Vite 7.0.4
- **Styling**: Tailwind CSS 4.1.11 + shadcn/ui components
- **Code Highlighting**: react-syntax-highlighter
- **State**: React hooks (useState)
- **Package Manager**: Bun 1.1.0
- **Code Quality**: ESLint (strict) + Prettier

## Project Structure
```
src/
├── components/
│   ├── interview/          # InterviewMaterial, QuestionCard
│   └── ui/                 # shadcn/ui components (badge, button, card, sonner, tabs)
├── services/
│   └── questionService.ts  # Question retrieval from JSON data
├── types/
│   └── interview.ts        # Question, QuestionCategory, CreateQuestionData, UpdateQuestionData
├── lib/
│   └── utils.ts            # Helper functions (cn for className merging)
├── App.tsx                 # Main component rendering InterviewMaterial
├── main.tsx                # Entry point
└── index.css               # Global styles + CSS variables

data/
├── data-structures.json    # Data structures questions (read-only)
├── low-level-design.json   # Low-level design questions (read-only)
└── high-level-design.json  # High-level design questions (read-only)
```

## Key Features Implemented

### 1. Interview Question Display
- Questions organized by three categories:
  - **Data Structures**: Algorithm and data structure problems
  - **Low Level Design**: Object-oriented design problems
  - **High Level Design**: Distributed systems and architecture problems
- Tabbed interface for easy navigation between categories
- Responsive design with mobile-friendly tab labels

### 2. Question Card Component
- Displays question title, description, difficulty, and tags
- Shows examples (if available)
- Syntax-highlighted code snippets using react-syntax-highlighter (vscDarkPlus theme)
- Formatted solution explanations
- Difficulty badges with color coding:
  - Easy: Green
  - Medium: Yellow
  - Hard: Red

### 3. Question Service
- Loads questions from three separate JSON files:
  - `data/data-structures.json` - Data structures questions
  - `data/low-level-design.json` - Low-level design questions
  - `data/high-level-design.json` - High-level design questions
- Combines all questions from the three files
- Provides methods to:
  - Get all questions
  - Get questions by category
  - Get question by ID
- Questions are read-only (create/update/delete operations throw errors)

### 4. UI/UX
- shadcn/ui components (modern, accessible)
- Responsive design (mobile-first)
- Toast notifications (Sonner) ready
- Dark mode ready (theme infrastructure exists)
- Syntax highlighting for code snippets

## Type Definitions
```typescript
type QuestionCategory = 'data-structures' | 'low-level-design' | 'high-level-design';

interface Question {
  id: string;
  title: string;
  description: string;
  category: QuestionCategory;
  difficulty: 'easy' | 'medium' | 'hard';
  tags: string[];
  solution?: string;
  examples?: string[];
  codeSnippet?: string;
  createdAt: string;
  updatedAt: string;
}
```

## Data Flow
1. **Questions**: `data/*.json` (3 files) → `questionService` (combines) → `InterviewMaterial` → `QuestionCard`
2. **Categories**: User selects tab → `InterviewMaterial` filters questions → displays in `QuestionCard` components
3. **Rendering**: `App.tsx` → `InterviewMaterial` → tabs → filtered questions → `QuestionCard`
